# Interface Contracts — Coordination Protocol

Interface contracts ensure parallel workers share a consistent API surface. The lead agent writes contracts before spawning workers; both producer and consumer receive the same spec.

---

## When a Contract Is Required

| Scenario | Contract? |
|----------|-----------|
| Parallel workers with shared interfaces | **Yes** |
| Sequential workers where later worker consumes earlier worker's output | Yes |
| Parallel workers operating on completely separate files | No |
| Single worker | No |

**Rule of thumb**: if Worker A exports something that Worker B imports, write a contract.

---

## Contract File Location

```
.tmp/work/<EPIC-ID>-interfaces.md
```

Create `.tmp/work/` if it does not exist. This directory is gitignored.

---

## Contract Template

```markdown
# Interface Contracts for <EPIC-ID>

Generated by Lead Agent on <date>.
All workers listed below must implement/consume these signatures exactly.

---

## <Interface Name>

- **Producer**: Task <TASK-ID>, file: `<path/to/file.ts>`
- **Consumer(s)**: Task <TASK-ID>, file: `<path/to/file.ts>`
- **Type**: hook | component | function | class | type | constant

### Signature

\`\`\`typescript
// Exact TypeScript signature — no paraphrasing
export function useUserProfile(userId: string): {
  profile: UserProfile | null;
  loading: boolean;
  error: Error | null;
}
\`\`\`

### Notes
- Any behavioral constraints not obvious from the signature
- Error handling expectations
- Null/undefined semantics

---

## Shared Types

\`\`\`typescript
// Types used by multiple workers — define here, one canonical source
export interface UserProfile {
  id: string;
  name: string;
  email: string;
  avatarUrl?: string;
}
\`\`\`

Shared types must be placed in: `src/types/<domain>.ts`
Both producer and consumer import from this path.
```

---

## Verification Checklist (Lead Agent)

After all workers complete, verify each contract:

1. **Producer exports match** — `grep -n "export" <producer-file>` — name and signature match contract
2. **Consumer imports resolve** — `bun run typecheck` — zero errors on import paths
3. **Shared types are in the right file** — not duplicated inline in components or hooks
4. **No extra exports added** — producer didn't add undocumented exports that consumers depend on

### Fixing deviations

| Deviation severity | Action |
|-------------------|--------|
| Import path wrong | Fix inline (lead agent edits the import) |
| Parameter renamed | Fix inline (lead agent adds alias or updates both sides) |
| Return type extended | Fix inline if additive and non-breaking |
| Structural redesign | Spawn a focused fix worker; block dependent tasks until resolved |

## Example: useUserProfile Contract

```markdown
# Interface Contracts for EPIC-7

## useUserProfile hook

- **Producer**: Task T-12, file: `src/hooks/useUserProfile.ts`
- **Consumer**: Task T-13, file: `src/components/ProfileCard.tsx`
- **Type**: hook

### Signature

\`\`\`typescript
export function useUserProfile(userId: string): {
  profile: UserProfile | null;
  loading: boolean;
  error: Error | null;
  refetch: () => void;
}
\`\`\`

### Notes
- Returns `null` for `profile` while loading or on error
- `refetch()` triggers a fresh API call; does not debounce
- Throws if `userId` is an empty string

## Shared Types

\`\`\`typescript
// src/types/user.ts
export interface UserProfile {
  id: string;
  name: string;
  email: string;
  avatarUrl?: string;
  role: "admin" | "member" | "viewer";
}
\`\`\`
```
